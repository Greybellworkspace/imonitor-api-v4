# Specification: Phase 3.2 — Core Features (Modules, Parameters, NodeDefinition)

**Track ID:** phase3.2-core-features_20260303
**Type:** Feature
**Created:** 2026-03-03
**Status:** Complete

## Summary

Migrate three v3 API modules — Modules, Parameters, and NodeDefinition — to NestJS v4. These modules provide system configuration management: Modules exposes menu/module metadata with report/widget-builder lookups; Parameters and NodeDefinition provide dynamic CRUD operations against configurable database tables (parameter tables and node definition tables respectively), including field-level encryption, Excel export, and permission-gated table access.

## Context

These are foundational modules that define the system's table metadata layer. `core_modules` defines the navigation/feature hierarchy, `core_modules_tables` maps modules to their database tables, and `core_tables_field` describes each table's columns. Parameters and NodeDefinition modules share nearly identical logic patterns — they differ only in filtering by `tableType` ('param' vs 'nodes') and target database (`iMonitorData`). Many later Phase 3.x modules (Reports, WidgetBuilder, Observability, Dashboard) depend on this metadata layer.

## Technical Notes

- **Parameters and NodeDefinition share ~90% logic.** Extract a shared `DynamicTableService` base that both services extend, differing only in `tableType` filter ('param' vs 'nodes').
- **Dynamic table queries** hit `iMonitorData` tables whose names are stored in `core_modules_tables.tableName`. These must use `LegacyDataDbService` raw SQL — NOT TypeORM.
- **Field encryption:** Password fields use MySQL `AES_ENCRYPT`/`AES_DECRYPT` with the key from `core_sys_config.aesEncryptionKey`. The encryption helper should be added to `SharedModule`.
- **Field visibility rules:** ID columns hidden+readonly, date/by columns visible+readonly, password columns hidden+editable, all others visible+editable.
- **User lookup for `_by` fields:** Dynamic queries JOIN `core_application_users` to resolve `created_by`/`updated_by` UUIDs to display names.
- **The Modules controller** delegates to services that query report/widget-builder join tables. Since those modules don't exist yet, implement the queries directly in `ModulesService` using TypeORM QueryBuilder on the existing entities.
